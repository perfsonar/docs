pSConfig Quickstart Guide¶

Note

This page contains a quick set of commands for publishing a pSConfig template and configuring the pScheduler and Grafana agents to use them. As the Quickstart in the title implies, this is not intended to be a comprehensive guid, for that see the rest of the pSConfig documentation.

On the host where the template is to be published…¶

  1. Make sure the perfsonar-psconfig-publisher package is installed:

    CentOS:

    yum install perfsonar-psconfig-publisher

    Debian/Ubuntu:

    apt-get install perfsonar-psconfig-publisher

  2. Create a pSConfig template.

  3. Publish the template:

    psconfig publish template.json

On each host running measurements…¶

  1. Make sure the perfsonar-psconfig-pscheduler package is installed:

    CentOS:

    yum install perfsonar-psconfig-pscheduler

    Debian/Ubuntu:

    apt-get install perfsonar-psconfig-pscheduler

  2. Run the following command to point the agent at the template:

    psconfig remote add https://MYHOST/psconfig/template.json

On your Grafana host…¶

  1. Make sure the perfsonar-psconfig-grafana package is installed:

    RedHat:

    dnf install perfsonar-psconfig-grafana

    Debian/Ubuntu:

    apt-get install perfsonar-psconfig-grafana

  2. Run the following command to point the agent at the template:

    psconfig remote add https://MYHOST/psconfig/template.json
